## PHASE 4: SECURITY & HARDENING (Days 6-7)

(A) Security audit checklist @security
    - [ ] No hardcoded passwords or API keys (grep -r "password\|secret\|key" cmd/)
    - [ ] SQL injection: All queries parameterized (check pkg/sqlx/)
    - [ ] SSRF: Spider validates URLs (check cmd/spider/handler/)
    - [ ] Rate limiting: Enabled per domain (check Spider config)
    - [ ] CORS: Frontend handles search requests securely
    - [ ] TLS: All external endpoints use HTTPS
    - [ ] Secrets: Use environment variables or secret store (not config files)

(B) Add circuit breaker pattern @resilience
    - Install: go get github.com/sony/gobreaker
    - Add to: cmd/conductor/handler/flow.go (Spider client)
    - Add to: cmd/frontend/grpc/client.go (Searcher client)
    - Configure: 3 failures → open circuit, 60s timeout

(B) Implement exponential backoff @resilience
    - Add to: cmd/spider/handler/flow.go (HTTP requests)
    - Retry up to 3 times with: 1s, 2s, 4s delays
    - Don't retry 4xx errors (permanent failures)

(B) Add comprehensive logging @operations
    - All major operations logged with slog.Info()
    - Errors logged with slog.Error() + stack trace
    - Sensitive data: never log passwords, tokens, API keys
    - Verify: No sensitive data in logs

(B) Verify graceful shutdown @resilience
    - Test: Kill running service with SIGTERM
    - Verify: In-flight requests complete (5-10s grace period)
    - Verify: gRPC server stops accepting new connections
    - Check logs: "Shutting down gracefully" message appears

(C) Document security model @documentation
    - Write: docs/SECURITY.md
    - Cover: data privacy, encryption, authentication model
    - Document known vulnerabilities and mitigations
